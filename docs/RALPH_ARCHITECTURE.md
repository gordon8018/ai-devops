## 1. 系统目标与设计原则

### 1.1 系统目标

Ralph 集成系统旨在为 ai-devops 提供一个强大的 AI 自主编码循环能力，实现从任务定义到代码生成、质量检查、Code Review、CI 监控的完整自动化流程。

**核心目标：**

1. **任务自动化执行**: 将高层次的 TaskSpec 转换为可执行的 PRD，驱动 Ralph 自动完成开发任务
2. **状态透明化**: 实时追踪任务执行状态，提供完整的进度可见性
3. **质量保障**: 通过质量门禁确保生成的代码符合质量标准
4. **知识协同**: 与知识库（Obsidian、gbrain）集成，为 AI 提供上下文增强
5. **反馈循环**: 持续收集执行数据，优化 AI 表现和流程效率

### 1.2 设计原则

| 原则 | 描述 |
|------|------|
| **单一职责** | 每个组件专注于一个核心功能，模块边界清晰 |
| **状态可恢复** | 所有中间状态持久化，支持从中断点恢复 |
| **可观测性** | 日志、指标、追踪贯穿全流程，便于调试和优化 |
| **可扩展性** | 支持多 Ralph 实例、多知识源、多种质量检查策略 |
| **安全性** | 隔离执行环境，防止 AI 操作影响系统稳定性 |

### 2.2 核心组件职责

| 组件 | 职责 |
|------|------|
| **Task Queue** | 接收和管理待执行任务，支持优先级和依赖 |
| **Task Manager** | 任务生命周期管理、调度、状态同步 |
| **State Store (ralph_state.py)** | 持久化任务状态、进度、日志 |
| **task_to_prd.py** | TaskSpec → prd.json 格式转换 |
| **Quality Gate** | 质量检查（Typecheck、Lint、Test）集成 |
| **ralph_runner.py** | Ralph.sh 执行包装器，超时和错误处理 |
| **Knowledge Sync** | 从知识库检索上下文，注入到 PRD |
| **Real-time Monitoring** | WebSocket 推送，Dashboard API |
| **Feedback Loop** | 质量评估，持续改进策略 |

```
┌──────────────┐
│  1. 创建任务  │
│  TaskSpec    │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  2. 转换 PRD │
│  TaskSpec →  │
│  prd.json    │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  3. 初始化    │
│  State Store │
└──────┬───────┘
       │
       ▼
┌──────────────┐
│  4. 上下文    │

└──────┬───────┘
       │
       ▼
┌──────────────┐

└──────┬───────┘
       │
       ▼
┌──────────────┐     ┌──────────────┐
│  6. 执行循环  │────▶│  7. 状态同步  │

       │                    │
       ▼                    ▼

       ▼
```

### 3.2 数据转换流

```
TaskSpec (ai-devops 格式)
{
  "taskId": "task-20260414-001",
  "task": "Add priority field",
  "acceptanceCriteria": [...],
  "userStories": [...]
}
        │
        │ task_to_prd.py
        ▼
prd.json (Ralph 格式)
{
  "project": "ai-devops",
  "branchName": "ralph/task-20260414-001",
  "userStories": [...],
  "qualityChecks": {...}
}
        │
        │ ralph.sh 执行
        ▼
progress.txt (进度跟踪)
{
  "iterations": 5,
  "stories": [
    {"id": "US-001", "passes": true}
  ],
  "progress_percent": 50
}
        │
        │ 状态同步
        ▼
ralph_state 表 (SQLite)
{
  "task_id": "task-20260414-001",
  "status": "running",
  "progress": 50,
  "logs": "Iteration 5 completed..."
}
```
